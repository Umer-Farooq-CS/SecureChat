# Test Reports Directory

This directory contains automatically generated test reports from both automatic and manual test suites.

## Report Types

### Automatic Test Reports
- **Format:** `automatic_tests_YYYYMMDD_HHMMSS.json` and `.txt`
- **Generated by:** `tests/run_all_tests.py`
- **Contains:**
  - Test summary (total, passed, failed, errors, skipped)
  - Success rate
  - Test modules executed
  - Detailed failure/error information
  - Skipped tests with reasons

### Manual Test Reports
- **Format:** `manual_<test_name>_YYYYMMDD_HHMMSS.json` and `.txt`
- **Generated by:** Manual test scripts
- **Test types:**
  - `manual_invalid_certificates_*.json` - Invalid certificate tests
  - `manual_tampering_*.json` - Message tampering tests
  - `manual_replay_*.json` - Replay attack tests
  - `manual_traffic_capture_*.json` - Wireshark capture tests

### Summary Reports
- **Format:** `test_summary_YYYYMMDD_HHMMSS.json` and `.txt`
- **Generated by:** `tests/manual/run_all_tests.sh`
- **Contains:** List of all generated reports

## Report Formats

### JSON Format
Structured data for programmatic access:
```json
{
  "report_type": "automatic_tests",
  "timestamp": "2025-11-13T11:44:45",
  "summary": {
    "total_tests": 88,
    "passed": 88,
    "failed": 0,
    "success_rate": 100.0
  },
  ...
}
```

### Text Format
Human-readable format for quick review and inclusion in documents.

## Usage

### View Reports
```bash
# View text report
cat reports/automatic_tests_*.txt

# View JSON report
cat reports/automatic_tests_*.json | python -m json.tool
```

### Include in Test Report Document
The text reports can be directly included in your test report document (`.docx`).

## Report Contents

Each report includes:
- **Summary:** Overall test statistics
- **Test Details:** Individual test results
- **Failures/Errors:** Detailed error information
- **Skipped Tests:** Reasons for skipped tests
- **Timestamps:** When tests were run

## Notes

- Reports are automatically generated after each test run
- Old reports are not automatically deleted (you may want to clean them up)
- Reports are saved in both JSON (structured) and TXT (human-readable) formats
- All timestamps are in ISO format
